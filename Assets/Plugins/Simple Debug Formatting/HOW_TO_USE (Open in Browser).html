Overview

The D class holds the public static Methods for the Debug Logs.

D.Log();
D.Warning();
D.Error();
D.Exception();

Log(), Warning() and Error() are called just as the normal Debug Logs, but Exception() returns an Exception and needs to be thrown in order to see a console output.

Presets

With the P enum you're able to save multiple format settings and apply them by just calling that enum entry.
Adding a new Preset
Go to the "Presets_SDF_SOB"-ScriptableObject, under "../Simple Debug Formatting/Resources/Presets_SDF_SOB.asset".
Click the "+" Button on the bottom right to create a new Preset.
After creation, choose a name for the Preset, set all settings to your liking and click the "Generate Presets" Button.

IMPORTANT
After removing a Preset from the List, "Generate Presets" will only remove the Preset from the P enum!
If you still have usages of that Preset in any of your scripts, you have to remove those yourself!
Messages
Since the Preset names will be written into the enum, they have to follow the enum naming conventions.
Whenever there's an error, the Message box will display what you have to change and also color the "Name" on the Preset that has the error, red.
You can only Generate the Presets when there're no errors!

The "Changes detected!" Message will be visible when the names in the inspector don't match with the enum entries in P.
(After adding a new Preset or renaming an existing one)
Alwyas click the "Generate Presets" Button when you see this Message! Otherwise not all Presets will work correctly.
